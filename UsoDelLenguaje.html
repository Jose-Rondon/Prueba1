Uso y aplicación del lenguaje de marcado

Atributos:

Los atributos HTML son palabras especiales que se utilizan dentro de la etiqueta de apertura para controlar el comportamiento del elemento. Los atributos HTML son un modificador de un tipo de elemento HTML. Un atributo modifica la funcionalidad predeterminada de un tipo de elemento o proporciona funcionalidad a ciertos tipos de elementos que no pueden funcionar correctamente sin ellos. En la sintaxis HTML, se agrega un atributo a una etiqueta de inicio HTML.

Los atributos HTML generalmente aparecen como pares nombre-valor, separados por =, y se escriben dentro de la etiqueta de inicio de un elemento, después del nombre del elemento:


Estructura:


<elemento atributo="valor">elementos del contenido</elemento>

Valores:

Los Valores de los atributos HTML, son los valores que se le asignan al paramentro dentro de la etiqueta.

Donde el elemento nombra el tipo de elemento HTML y el atributo es el nombre del atributo, establecido en el valor proporcionado. El valor puede estar entre comillas simples o dobles, aunque los valores que constan de ciertos caracteres pueden dejarse sin comillas en HTML (pero no en XHTML).Dejar valores de atributo sin comillas se considera inseguro.

Aunque la mayoría de los atributos se proporcionan como pares de nombres y valores, algunos afectan al elemento simplemente por su presencia en la etiqueta de inicio del elemento.

Atributos comúnes:

ID


El atributo id proporciona un identificador único en todo el documento para un elemento. Esto se puede utilizar como selector de CSS para proporcionar propiedades de presentación por el navegador para centrar la atención en el elemento específico, o por scripts, para alterar el contenido o la presentación de un elemento.

Anexada a la URL de la página, la URL apunta directamente al elemento específico dentro del documento, generalmente una subsección de la página.

Por ejemplo, el ID "format" en este documento HTML está siendo usado para dar formato a diferentes partes del mismo.


Ejemplo:


	<!DOCTYPE html><html>
		<head>
			<meta chatset="utf-8"/>
			<title>Descripción en la Pestaña</title>
			<style>
				#ejemplo { <---- ID en el Documento
				font-family:sans-serif;
				font-size:14px;
				color:white;
				}
			</style>
		</head>
		<body>
			<h1>Título</h1>
			<p>Contenido</p>
		</body>
	</html>

CLASS


El atributo de class proporciona una forma de clasificar elementos similares. Esto se puede utilizar con fines semánticos o con fines de presentación. Semánticamente, por ejemplo, las classs se utilizan en microformatos. Presentacionalmente, por ejemplo, un documento HTML puede usar la designación class = "notation" para indicar que todos los elementos con este valor de class están subordinados al texto principal del documento.

Estos elementos pueden reunirse y presentarse como notas a pie de página en una página en lugar de aparecer en el lugar donde aparecen en la fuente HTML. Otro uso de la presentación sería como selector de CSS.


Ejemplo:


	<!DOCTYPE html><html>
		<head>
			<meta chatset="utf-8"/>
			<title>Descripción en la Pestaña</title>
			<style>
				.ejemplo { <---- Class en el Documento
				font-family:sans-serif;
				font-size:14px;
				color:white;
				}
			</style>
		</head>
		<body>
			<h1>Título</h1>
			<p>Contenido</p>
		</body>
	</html>

STYLE


Un programador puede usar las propiedades de presentación de los códigos no atribuibles de style a un elemento en particular. El atributo de style se puede usar en cualquier elemento HTML (se validará en cualquier elemento HTML; sin embargo, no es necesariamente útil). Se considera una mejor práctica usar la identificación de un elemento o los atributos de class para seleccionar el elemento con una hoja de style, aunque a veces esto puede ser demasiado engorroso para una aplicación simple y específica o ad hoc de propiedades con style.


Ejemplo:


	<!DOCTYPE html><html>
		<head>
			<meta chatset="utf-8"/>
			<title>Descripción en la Pestaña</title>
			</head>
		<body style="font-family:sans-serif"> <- style en el documento
			<h1>Título</h1>
			<p>Contenido</p>
		</body>
	</html>

TITLE


El atributo de title se utiliza para adjuntar una explicación subtextual a un elemento. En la mayoría de los navegadores, este atributo se muestra como lo que a menudo se denomina información sobre herramientas.

No se puede tener mas the un "title" en un documento HTML.


Ejemplo:


	<!DOCTYPE html><html>
		<head>
			<meta chatset="utf-8"/>
			<title>Descripción en la Pestaña</title>
			title en el documento
			</head>
		<body>
			<h1>Título</h1>
			<p>Contenido</p>
		</body>
	</html>

Enlaces y tipos de enlaces (internos y externos)

HREF:


  • Para los elementos <a> y <area>, el atributo href especifica la URL de la página a la que va el enlace.

  • Para los elementos <base>, el atributo href especifica la URL base para todas las URL relativas en una página.

  • Para los elementos <link>, el atributo href especifica la ubicación (URL) del recurso externo (generalmente un archivo de hoja de estilo).


    • Se aplica a:


    Elementos Atributo
    <a> <href>
    <area> <href>
    <base> <href>
    <link> <href>

    Ejemplo de href


    <a href>="miDocumento.docx">Leer documento</a>


    NAME:


  • El atributo de nombre especifica un nombre para un elemento HTML.

  • Este atributo de nombre se puede utilizar para hacer referencia al elemento en un JavaScript.

  • Para un elemento <form>, el atributo de nombre se usa como referencia cuando se envían los datos.

  • Para un elemento <iframe>, el atributo de nombre se puede usar para apuntar a un envío de formulario.

  • Para un elemento <map>, el atributo name está asociado con el atributo usemap de <img> y crea una relación entre la imagen y el mapa.

  • Para un elemento <meta>, el atributo de nombre especifica un nombre para la información / valor del atributo de contenido.

  • Para un elemento <param>, el atributo name se usa junto con el atributo value para especificar parámetros para el complemento especificado con la etiqueta <object>


    • Ejemplo:


    <iframe src="miDocumento.html" name="Página2" width="50%" height="500" style="border:3px solid blue;"></iframe>


    Direcciones Relativas:


    Una ruta relativa siempre es relativa a la raíz del documento, por lo que si el archivo html está en el mismo nivel del directorio, deberá iniciar la ruta directamente con el nombre de su directorio de imágenes.


    Ejemplo:


    <a href="imagenes/imagen.png">


    Direcciones Absolutas


    Una ruta absoluta "/" apunta la raíz del sitio no a la raíz del documento: ~/ en nuestro caso. Es por eso que necesita especificar la carpeta de su documento para acceder a la carpeta de imágenes:


    Ejemplo:


    <a href="~/.../imagenes/imagen.png">


    Enlaces a correos electrónicos (mailto)


    Se usa mailto: dentro del atributo href para crear un enlace que abra el programa de correo electrónico del usuario (para permitirle enviar un nuevo correo electrónico)


    Ejemplo:


    <a href="mailto:micorreo@gmail.com">Enviar correo</a>

    En el ejemplo establecido se indica en caso de que el usuario haga click en el link este abrirá el cliente de correo electrónico predeterminado en su computador.

    Insertar Imágenes

    Atributos de Imágenes


  • La etiqueta <img> se utiliza para incrustar una imagen en una página HTML.

  • Las imágenes no se insertan técnicamente en una página web; las imágenes están vinculadas a páginas web. La etiqueta <img> crea un espacio de espera para la imagen referenciada.

  • La etiqueta <img> tiene dos atributos obligatorios:
  • src: especifica la ruta a la imagen.
  • alt: especifica un texto alternativo para la imagen, si la imagen por algún motivo no se puede mostrar.

  • Siempre especificar el ancho y el alto de una imagen. Si no se especifican el ancho y el alto, la página puede parpadear mientras se carga la imagen.

  • Para vincular una imagen a otro documento, simplemente anide la etiqueta <img> dentro de una etiqueta <a>.


    • Ejemplo


    <img src="imagenes/imagen.png" alt="imagen.png" width="50px" height="50px" style="vertical-align:middle">


    Alineación y Posicionamiento de Imágenes


    La posición de las imágenes puede ser modificada mediante sus atributos como <width>, <height> y <style>.


    Ejemplo:


    <img src="imagenes/imagen.png" alt="imagen" width="50px" hieght="50px" style="float:left">

    Texto Alternativo


    Es el texto que se muestra en caso que la imagen no sea cargada.


    Ejemplo:


    <img src="imagenes/imagen.png" alt="Error en la carga">


    Tamaño y Resolución Adecuados


    Imágenes de ancho completo:


    En todo sitio web, es posible existan secciones en las que se necesite usar imágenes de ancho completo que cubran toda la pantalla de izquierda a derecha. Estas áreas pueden ser imágenes de héroe, presentaciones de diapositivas en pantalla completa, imágenes de banner. Para asegurar que las imágenes de ancho completo se vean bien en cualquier dispositivo, grande o pequeño, el tamaño recomendado es 2400x1600px96dpi. Se debe tener en cuenta que los dispositivos tienen un aspect:ratio diferente a la que se espera y es posible que las imágenes se recorten cuando se vean en la web. Como recomendación, se debe intentar realizar tomas más amplias de lo habitual para el contenido web, para evitar recortar el contenido importante de la imagen.

    Imágenes dentro de contenido:


  • Horizontal: ancho máximo 1500px, jpg, guardado para web y optimizado (96dpi).

  • Vertical: ancho máximo 1000px, jpg, guardado para web y optimizado (96dpi).
    • En las páginas web, habrá secciones que consisten en imágenes, áreas de texto, botones de llamada a la acción, etc. Básicamente, estas son las secciones en las que las imágenes son parte del contenido interno. En estos casos, a menos que se especifique el tamaño recomendado, se pueden seguir las recomendaciones anteriores para imágenes horizontales y verticales. Según su proporción, la altura se ajustará en consecuencia cuando se guarden las imágenes.


    Galería de Diapositivas


  • Si se están preparando imágenes para las galerías de presentación de diapositivas hay que asegúrese que todas las imágenes tengan la misma altura recomendada de 1500px,96dpi independientemente de si son horizontales o verticales.
